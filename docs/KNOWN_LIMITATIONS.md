# Known limitations

This repository is an implemented foundation, not a production-complete release. The core identity, tenant-scoped case, private-file, audit, outbox, migration, and bilingual web foundations are present. The following gaps prevent a production launch and prevent the repository from meeting the full product definition of done.

## External inputs required before production

- Hosting account, DNS/TLS, network boundaries, managed PostgreSQL/Redis/S3, secret manager, backup/PITR policy, monitoring destinations, and approved RPO/RTO.
- Stripe live account, live/restricted secret and webhook signing secret, production webhook URL, settlement/refund ownership, and final VND/USD commercial configuration.
- Stripe Connect enablement, supported-country/currency and payout/compliance ownership, approved clinic onboarding terms, and Connect acceptance testing.
- Approved external calendar synchronization service, provider OAuth applications, scoped credentials, busy-window/webhook/reconciliation contract, and provider-backed acceptance testing.
- Transactional email account, verified sending domain, DKIM/SPF/DMARC, credentials, suppression/bounce handling, and approved templates.
- Production object-storage credentials/policy and malware-scanning capacity/signature update process.
- Approved OpenTelemetry collector and error-tracking project with retention and sensitive-data scrubbing.
- Optional SMS/realtime messaging providers and an approved video provider. The bounded manual meeting-link adapter remains fail-closed unless explicitly configured.
- Jurisdiction-specific privacy/medical/legal review, data-processing contracts, cross-border transfer mechanism, record-category retention schedule, breach-notification duties, and final consent/notices.
- Compliance-approved clinic/dentist verification checklist, evidence sources, reviewer conflict/appeal policy, badge wording, and operational reviewers.

Production configuration rejects missing/development authentication and encryption secrets, local endpoints/credentials, insecure application origins, insecure or unauthenticated SMTP, partially configured/insecure optional messaging providers, a non-Stripe payment adapter, missing Stripe signing material, non-live Stripe secret keys, a development meeting adapter, an empty meeting-host allowlist, or a development/non-HTTPS calendar adapter.
